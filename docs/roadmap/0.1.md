# Roadmap — 0.1

Milestones for the first public release (`0.1.0`). Each milestone follows the spec-driven loop: write the spec docs first, then the code, then the tests.

A milestone is **done** when: (a) its spec docs exist in `docs/`, (b) the code implements them, (c) tests pass, (d) the playground (where applicable) demonstrates the feature end-to-end.

## M1 — Core

**Goal:** `@choo-choo/core` delivers the IR, the manual builder, and the SVG renderer.

**Specs required first:**
- `docs/ir.md` — the `Node` discriminated union (every kind, every field).
- `docs/rendering.md` — the renderer's visitor contract and the shape of the emitted SVG.
- `docs/builder.md` — the manual API: factory functions, ergonomics, examples.

**Deliverables:**
- `packages/core` with IR, builder, renderer, and tests.
- Repo-level tooling: `pnpm-workspace.yaml`, `tsconfig.base.json`, `biome.json`, root `package.json`.
- Inline snapshot tests covering every IR node kind.

**Done when:** manual construction produces a deterministic SVG string via `pnpm --filter @choo-choo/core test`.

## M2 — EBNF parser

**Goal:** `@choo-choo/parser-ebnf` turns ISO/IEC 14977 EBNF source into IR.

**Specs required first:**
- `docs/grammars/ebnf.md` — supported EBNF subset with examples (ported from the legacy project's parser docblocks).

**Deliverables:**
- `packages/parser-utils` with reader, tokenizer primitives, and base specification (created here because EBNF is the first parser to need them).
- `packages/parser-ebnf` implementing `GrammarParser`.
- Separate tokenizer and parser tests plus snapshot fixtures.

**Done when:** feeding the legacy project's test fixtures through the new parser produces IR whose rendered SVG matches inline goldens.

## M3 — Vanilla binding + playground

**Goal:** a working vertical slice — user types EBNF in a playground, a `<choo-choo>` custom element renders the diagram.

**Specs required first:**
- `docs/bindings/vanilla.md` — `<choo-choo>` attributes, `mount()` signature, dynamic-import behaviour for parsers.
- `docs/composition.md` — the `compose` transform that inlines prior rules into a rendered rule.

**Deliverables:**
- `packages/vanilla` with the custom element and the imperative `mount()` helper.
- `composeRule()` in `@choo-choo/core`, surfaced through the vanilla binding as the `compose` attribute/option. Lives in core so future bindings inherit it for free.
- `apps/playground` (Vite) demoing both manual and EBNF paths plus the three compose modes. ANTLR and PEG tabs are added in their respective milestones.

**Done when:** `pnpm --filter playground dev` serves a page where manual IR and EBNF source both render, and switching the `compose` control re-renders the selected rule with prior rules inlined.

## M4 — React binding

**Goal:** `@choo-choo/react` exposes a React component usable in server and client contexts.

**Specs required first:**
- `docs/bindings/react.md` — props, SSR behaviour, CSS import.

**Deliverables:**
- `packages/react` with the `<ChooChoo>` component and happy-dom tests covering both client and SSR paths.
- `styles.css` relocated from `@choo-choo/vanilla` to `@choo-choo/core`; both bindings re-export it via a one-line `@import`. All future bindings follow the same pattern.

**Deferred (not part of M4):**
- Playground React tab — the playground keeps using the vanilla binding only for now. Revisit after M7 when the docs site and all bindings are in.

**Done when:** the React component renders on the server with no `"use client"` directive (verified via `renderToStaticMarkup`) and updates reactively on the client (verified via happy-dom).

## M5 — ANTLR parser

**Goal:** `@choo-choo/parser-antlr` adds ANTLR4 grammar support, validating the parser contract against a second grammar.

**Specs required first:**
- `docs/grammars/antlr.md` — supported ANTLR4 subset: rule syntax (`name : ... ;`), cardinalities (`?`, `*`, `+`), lexer vs parser rules, labels, token references, comments.

**Deliverables:**
- `packages/parser-antlr` implementing `GrammarParser`, reusing `parser-utils`.
- Fixtures drawn from real-world ANTLR grammars (e.g. C# spec fragment).
- Playground exposes a grammar selector (EBNF / ANTLR).

**Done when:** both parsers coexist in the playground and the same rendering path accepts either.

## M6 — PEG parser

**Goal:** `@choo-choo/parser-peg` adds PEG support, exercising ordered-choice and lookahead semantics.

**Specs required first:**
- `docs/grammars/peg.md` — supported PEG subset: ordered choice (`/` or `|` depending on dialect), lookahead predicates (`&`, `!`), cardinalities, grouping, character classes. Document how PEG semantics are expressed in the IR (e.g. whether ordered choice is a distinct `node.kind` or a flag on `choice`).

**Deliverables:**
- `packages/parser-peg` implementing `GrammarParser`.
- Fixtures drawn from Python's PEG grammar (PEP 617).
- Playground exposes PEG alongside EBNF and ANTLR.

**Done when:** all three launch parsers coexist and the playground can switch between them at runtime.

## M7 — Vue + Astro bindings + docs site

**Goal:** complete framework coverage and stand up the public docs site.

**Specs required first:**
- `docs/bindings/vue.md`.
- `docs/bindings/astro.md`.

**Deliverables:**
- `packages/vue` (Vue 3 SFC) and `packages/astro` (`.astro` component).
- `apps/docs` (Astro Starlight) wiring the specs into a browsable site.

**Done when:** every binding has a live example in `apps/docs`.

## M8 — Release prep

**Goal:** publish `0.1.0`.

**Deliverables:**
- Changesets configured for per-package versioning.
- GitHub Actions CI running Biome + Vitest + tsup build on Node 20/22.
- Release docs, `LICENSE`, and contribution guide.

**Done when:** packages are published to npm and `apps/docs` is deployed.
